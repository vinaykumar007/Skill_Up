## 🧠 Why do we need OOP?

* Helps organize code into reusable structures
* Makes code modular and easier to maintain
* Models real-world entities using objects
* Reduces code duplication

---

## 🧱 Class & Object

### 🔹 Class

A blueprint for creating objects.

### 🔹 Object

An instance of a class.

### 🔹 Attributes

Variables that store object data.

### 🔹 Methods

Functions that define object behavior.

---

## 🔄 Procedural vs OOP

| Feature     | Procedural Programming | OOP                 |
| ----------- | ---------------------- | ------------------- |
| Structure   | Functions              | Classes & Objects   |
| Reusability | Limited                | High                |
| Maintenance | Difficult (large code) | Easier              |
| Approach    | Step-by-step           | Real-world modeling |

## 🚀 Why OOP for Large-Scale Applications?

* Code is modular and scalable
* Easier debugging and testing
* Supports team collaboration
* Promotes reusability (DRY principle)

---

## 🧠 Memory: Stack vs Heap

### 🔹 Stack Memory

* Stores function calls
* Fast access
* Temporary data

### 🔹 Heap Memory

* Stores objects
* Slower than stack
* Used for dynamic memory allocation
